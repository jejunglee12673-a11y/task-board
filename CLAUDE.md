## Git 運用ルール

### 基本方針

- コードを変更するたびに、必ず GitHub にプッシュすること
- コミットは意味のある単位でまとめ、変更内容が明確に伝わるメッセージを書くこと
- `main` ブランチへの直接プッシュは許可する

### コミット・プッシュ手順

コードを変更した後は、必ず以下を実行する：

```bash
git add <変更ファイル>
git commit -m "変更内容の説明"
git push origin main
```

### コミットメッセージ規則

- 日本語・英語どちらでも可
- 変更の「何を」「なぜ」が伝わる内容にする
- 例：`feat: タスク追加フォームを実装`、`fix: 完了フラグが保存されないバグを修正`

## 開発ガイドライン

- コメントは原則書かない。書く場合は「なぜ」が自明でない箇所のみ
- セキュリティ上の脆弱性（XSS、SQLインジェクション等）を導入しない
- 不要な抽象化・将来の要件を見越した過剰設計はしない
